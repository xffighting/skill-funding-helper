# UI Redesign Notes

The previous concept was useful but too repository-shaped. The redesign makes the first impression clearer for readers who do not yet know why a funding helper matters.

## Main changes

- Lead with the maintainer problem instead of a command.
- Show generated files as a concrete output map.
- Keep the palette quiet, with green for funding, blue for validation, yellow for launch copy, and red only for safety warnings.
- Place commands below the visual explanation.
- Add screenshot-ready sections so the README, GitHub social preview, and WeChat article can reuse the same visual language.

## UI structure

1. Hero with project name, one-line promise, and two commands.
2. Output rail showing FUNDING, README block, config, validator, and launch kit.
3. Before and after panel for the repo experience.
4. Safety boundary band.
5. Article and docs links.
